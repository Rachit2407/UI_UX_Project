UX Design Report CSE3252 · Rachit Yadav · 23FE10CSE00601
🧠 Design Thinking Model Applied
1
Empathise
Analysed 6 real EdTech platforms (Duolingo, Khan Academy, Udemy, Coursera, Unacademy, Byju's) to understand what learners actually experience — friction, dark patterns, cognitive overload.
→
2
Define
Problem: EdTech platforms prioritise retention metrics over learning outcomes, causing anxiety, drop-off and distrust through dark patterns.
→
3
Ideate
Explored Focus Mode, ethical streak systems, 1-tap resume, transparent pricing and distraction-free video player as solutions to each identified flaw.
→
4
Prototype
Built this fully interactive HTML/CSS/JS prototype — functional auth, video player, quiz engine, leaderboard, settings, and per-user data persistence.
→
5
Test
Heuristic evaluation against Nielsen's 10 principles, usability tested for error prevention, feedback loops, user control, and accessibility for diverse users.
📊 Case Study — 6 EdTech Platforms Analysed
🥇 Category 1 — Gold Standard
Duolingo
✅ Hook Model mastery — ethical habit loops via streak + variable rewards. Loss aversion used responsibly.
Khan Academy
✅ Zero ads, zero upsells. Mastery Tower + flow state. Minimalist interface = pure content focus.
🥈 Category 2 — Average (Functional but Flawed)
Udemy
⚠️ Banner blindness from fake urgency ("Sale ends in 5 hours"). Information overload violates Miller's Law.
Coursera
⚠️ Desktop-first web wrapper. "Audit for free" hidden (dark pattern). Weekly locks break self-paced learning.
🥉 Category 3 — Worst (UX Friction)
Unacademy
❌ Notification spam during class. Split-attention violation. Prevents flow state entirely.
Byju's
❌ Roach motel design. Mid-lesson paywalls. Revenue-first hierarchy destroys trust and learning continuity.
👥 Diverse User Considerations
👴
Age — Older Users
Large readable fonts (15px base), high contrast dark theme, clear label text on all buttons, no time-pressured interactions, simple 5-item navigation.
♿
Disability — Accessibility
All interactive elements have aria-label attributes, keyboard navigation (Space/Arrow/M/F shortcuts), camelCase IDs for screen readers, focus rings on inputs.
📱
Tech Literacy — Beginners
Familiar dashboard patterns (Jakob's Law), progressive disclosure hides complexity, onboarding flow limits choices (Hick's Law — 4 options max), tooltips on every icon.
🌐
Language & Literacy
Plain English throughout, no jargon in UI labels, icon + text combinations (never icon-only), Settings supports English/Hindi/Gujarati language toggle.
📶
Low Bandwidth / Offline
No external CDN dependencies — entire app is a single self-contained HTML file. All fonts loaded from Google Fonts (gracefully degrades). localStorage for data persistence.
📐
Mobile / Small Screens
Fully responsive at 800px and 520px breakpoints. Hamburger sidebar for mobile, stacked grid layouts, larger touch targets, collapsible sections.

🧪 Usability Testing Summary
Scenario 1 — New User Onboarding
Task: Sign up and reach the first lesson within 3 taps.
✓ Result: Sign up → Onboarding (2 steps) → Dashboard → Resume = 3 taps. Time-to-Learning minimised.
Scenario 2 — Error Recovery
Task: Enter wrong password and recover.
✓ Result: Form shakes, fields turn red, clear error message shown. User can immediately retry or use "Forgot password".
Scenario 3 — Focus Mode
Task: Hide all gamification elements to study without distraction.
✓ Result: Single click on "Enter Focus" hides streaks, badges, leaderboard. One click to restore. Zero cognitive overhead.
Scenario 4 — Course Discovery
Task: Find and enrol in a Data Science course.
✓ Result: Explore → filter "Data Science" → Enrol Free. No fake urgency, no upsell banners. 2 taps.
Scenario 5 — Multi-user Switch
Task: Log out and log in as different user.
✓ Result: Logout clears all state. Demo accounts load in 1 click. Each user has isolated XP, progress and notes.
Scenario 6 — Accessibility
Task: Navigate entire app using only keyboard.
✓ Result: / for search, Space to play, ← → for lessons, M to mark done, F for focus, Esc to close overlays.
